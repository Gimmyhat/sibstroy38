# Обработка изображений

Скрипт `process-images.sh` берёт оригиналы из `/c/Projects/Sibstroy38/img-src/`
и кладёт оптимизированные WebP в `deploy/img/`.

## Когда запускать

- Заказчик прислал новые фото объектов → положить их в `img-src/renders/` или
  создать `img-src/objects/` и запустить скрипт.
- Получены новые планировки или листы АР → положить в соответствующую папку
  `img-src/plans/` или `img-src/process/`.
- Нужно изменить параметры обработки (контраст, sharp, размер) → редактировать
  скрипт, запустить заново.

## Что делает по группам

| Папка | Стратегия |
|---|---|
| `renders/` (фасады) | Resize 1920px, auto-level, насыщенность +18%, sharp, WebP q84 |
| `plans/` (3D-планы) | Trim полей, resize 1600px, насыщенность +12%, WebP q88 |
| `constructs/*.png` (с подписями материалов) | Trim, resize 1600px, осторожный sharp, WebP q90 |
| `constructs/*.jpg` (3D без подписей) | Больше контраст и насыщенность, WebP q84 |
| `process/` (чертежи АР) | Только resize 1600px + WebP q82 — без визуальных правок (чертёж должен оставаться чертежом) |

## Запуск

```bash
bash img-tools/process-images.sh
```

Требует установленного ImageMagick 7.x по пути `/c/Program Files/ImageMagick-7.1.2-Q16-HDRI/`.
Установка на Windows: `winget install ImageMagick.ImageMagick`.

## Что НЕ делать

- Не коммитить `img-src/` (оригиналы) — он вне `deploy/` и в `.gitignore`.
- Не править `deploy/img/*.webp` руками — они перезаписываются скриптом.
- Не менять параметры обработки `process/*` без причины — чертежи должны
  выглядеть как чертежи, любые улучшения «красивости» снижают доверие.
